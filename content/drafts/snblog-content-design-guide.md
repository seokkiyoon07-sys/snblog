# SN Blog Content Design Guide

## 목적

이 문서는 `snblog`에서 발행하는 블로그 글의 시각적 일관성을 맞추기 위한 내부 작성 가이드다.
기준은 현재 실제 코드와 운영 중인 콘텐츠 구조를 바탕으로 잡는다.

- 폴더 구조 기준: `content/startup`, `content/columns`, `content/datalab`, `content/problem-download`, `content/originals`, `content/notice`
- 공통 본문 렌더링 기준: `ArticleContent`, `BlogLayout`, `globals.css`
- 카드/목록 노출 기준: `PostCard`
- 공통 보조 UI 기준: `Callout`, `TOC`

이 문서의 목표는 "예쁜 글"보다 "같은 팀이 만든 글처럼 보이는 글"을 만드는 것이다.

---

## 1. 현재 블로그에서 실제로 보이는 디자인 문법

### 1-1. 전체 인상

현재 블로그의 가장 강한 인상은 아래 4가지다.

1. 교육기관다운 차분함
2. 정보 전달 중심의 높은 가독성
3. 과장되지 않은 카드형 레이아웃
4. 이미지, callout, 태그, 메타 정보를 이용한 구조화

즉, 이 블로그는 잡지형 감성보다는 "정리 잘 된 교육 콘텐츠 허브"에 가깝다.
앞으로도 이 톤을 유지하는 것이 좋다.

### 1-2. 컬러 톤

실제 전역 색상 기준의 중심은 SN 딥그린 계열이다.

- Primary: `#122918`
- Primary light: `#1a3d2a`
- Primary dark: `#0f1f15`

운영 원칙:

- 브랜드 강조색은 `sn-primary` 계열 하나를 기본 축으로 쓴다.
- 본문에서는 브랜드색을 남발하지 않는다.
- 본문은 회색/검정 위주, 강조는 필요한 지점만 딥그린이나 정보성 callout 색으로 준다.
- 링크나 카테고리, 활성 상태 정도만 브랜드색을 쓰는 것이 현재 구조와 잘 맞는다.

### 1-3. 타이포그래피

현재 코드는 가독성 중심이다.

- 기본 산세리프 본문
- 한글 줄바꿈 보호: `word-break: keep-all`
- 본문은 넉넉한 행간
- 제목은 굵고 명료하며 장식이 약함

운영 원칙:

- 본문은 꾸미지 말고 읽히게 쓴다.
- 제목은 길어도 되지만, 한 문장에서 핵심 메시지가 바로 보여야 한다.
- 한글 문장은 짧게 끊고, 문단은 2~4문장 단위로 유지한다.
- 모바일에서 어색한 줄갈림이 생기지 않도록 지나치게 긴 한 줄 문장은 피한다.

### 1-4. 간격과 밀도

현재 공통 컴포넌트는 모두 "여백이 충분한 편"이다.

- 카드 라운드가 큼
- 이미지와 본문 사이 간격이 넉넉함
- 태그, 메타, 요약이 덩어리별로 분리됨
- TOC, callout, 카드 모두 숨 쉴 공간이 있음

운영 원칙:

- 정보량이 많아도 빽빽하게 보이면 안 된다.
- 섹션은 명확히 나누고, 중요한 블록 앞에는 여백을 확보한다.

## 2. 카테고리별 디자인 방향

폴더를 나눈 현재 구조는 매우 좋다. 다만 폴더가 다르더라도 "같은 블로그"라는 공통 인상은 유지해야 한다.

### 2-1. `startup`

주제:

- SNarGPT
- AI 기능 업데이트
- 기술 방향성
- 제품/서비스 철학

디자인 톤:

- 가장 현대적이고 명확한 톤
- 정보 박스, 기능 요약, before/after, 프로세스 설명이 잘 어울림
- 이미지 캡처, UI 예시, 기능 흐름도 활용도가 높음

권장 요소:

- 핵심 업데이트 요약 박스
- "무엇이 바뀌었나" 3~5개 리스트
- 사용 예시 이미지
- FAQ 또는 사용 시나리오

주의:

- 기술 글이라고 지나치게 개발자 문체로 가지 않는다.
- 교육 현장에 어떤 변화가 있는지까지 연결해야 SN 브랜드와 맞다.

### 2-2. `columns`

주제:

- 교육 철학
- 공부법
- AI와 교육에 대한 해석
- 브랜드 관점의 에세이형 글

디자인 톤:

- 가장 글맛이 살아도 되는 영역
- 그러나 레이아웃은 여전히 절제되어야 함
- 제목, 리드문, 중간 소제목의 힘이 중요함

권장 요소:

- 강한 도입 2~3문단
- 논지 전환이 분명한 H2 구조
- 문장 강조용 quote/callout

주의:

- 감성적인 표현은 가능하지만 화면은 미니멀하게 유지
- 이미지가 없어도 글이 버티는 구조여야 함

### 2-3. `datalab`

주제:

- 데이터 비교
- 맵
- 인포그래픽
- 분석형 콘텐츠

디자인 톤:

- 가장 실용적이고 분석적인 톤
- 표, 지도, 요약 지표, 해석 포인트가 중요함

권장 요소:

- 결론 먼저
- 데이터 해석 bullet
- 표/지도/캡처
- "이 데이터를 어떻게 봐야 하는가" 안내

주의:

- 데이터를 보여주는 것만으로 끝내지 말고, 해석 레이어를 반드시 붙인다.

### 2-4. `problem-download`

주제:

- 문제 자료
- 해설
- 다운로드
- 학습 활용법

디자인 톤:

- 가장 기능적이어야 함
- 실용성과 즉시성이 최우선

권장 요소:

- 자료 성격 요약
- 대상 학년/난이도/활용법
- 다운로드 또는 미리보기 유도
- 주의사항과 사용 팁

주의:

- 서론이 길면 안 됨
- 사용자가 "이 자료가 나한테 필요한가"를 10초 안에 판단할 수 있어야 함

### 2-5. `originals`

주제:

- 고전문학
- 작품 해설
- 작품 몰아보기

디자인 톤:

- 가장 교육 콘텐츠다운 형식미가 필요함
- 본문 가독성과 구조화가 중요함
- 설명, 원문, 해석, 포인트 구분이 분명해야 함

권장 요소:

- 작품 개요
- 핵심 장면 또는 핵심 구절
- 시험 포인트 정리
- 현대어 풀이/주제/표현상 특징 블록

주의:

- 화려한 시각 효과보다 학습 흐름이 우선
- 섹션 제목을 일관되게 반복해도 오히려 좋음

### 2-6. `notice`

주제:

- 공지
- 일정
- 입시안내
- 운영 안내

디자인 톤:

- 가장 명확하고 업무적이어야 함
- 감성보다 정확성이 우선

권장 요소:

- 핵심 일정 요약
- 대상/적용 범위
- 신청/확인 방법
- Q&A 또는 유의사항

주의:

- 문장 장식보다 정보 우선순위가 중요함
- 날짜, 시간, 대상은 시각적으로 눈에 띄어야 함

## 3. 글 한 편의 기본 구조

카테고리가 달라도 아래 순서를 기본 골격으로 가져가면 대부분 안정적이다.

1. 제목
2. 한 줄 요약 또는 리드
3. 메타 정보
4. 대표 이미지 또는 핵심 시각자료
5. 도입
6. 본문 섹션 3~6개
7. 요약 또는 다음 행동 유도

권장 템플릿:

```md
# 제목

한 줄 요약 또는 독자가 이 글에서 얻는 것

## 왜 이 글이 필요한가

문제 상황, 배경, 독자 맥락

## 핵심 내용 1

설명

## 핵심 내용 2

설명

## 이렇게 활용하면 좋다

실전 팁, 적용 포인트

## 마무리

핵심 요약, 다음 행동 유도
```

## 4. 제목, 요약, 메타 규칙

### 4-1. 제목

좋은 제목의 조건:

- 핵심 키워드가 앞에 보인다
- 독자가 얻는 이익이 드러난다
- 너무 광고처럼 보이지 않는다

권장:

- `SNarGPT 국어 RAG 16,000문항 업데이트`
- `AI로 공부하면 왜 격차가 더 벌어질까`
- `2026 입시안내: 꼭 확인해야 할 변경사항`

비권장:

- 너무 추상적인 제목
- 감탄사 위주 제목
- 키워드가 뒤에만 숨어 있는 제목

### 4-2. 요약문(excerpt)

역할:

- 카드 목록에서 클릭을 유도
- 제목만으로 부족한 맥락 보완

원칙:

- 2문장 이내
- 90~150자 정도
- "무엇을 알려주는 글인지" 바로 보이게

### 4-3. 메타 정보

현재 카드 구조상 아래 정보가 핵심이다.

- 날짜
- 읽는 시간
- 카테고리
- 태그

운영 원칙:

- 태그는 3~5개가 적당
- 읽는 시간은 실제 체감과 너무 다르지 않게
- 카테고리와 태그가 중복 의미만 가지지 않도록 조정

## 5. 본문 스타일 규칙

### 5-1. 문단

- 한 문단은 2~4문장 권장
- 모바일 기준 5줄 이상 길어지면 분리 검토
- 중요한 문장은 독립 문단으로 뽑아도 좋음

### 5-2. 소제목

- H2는 글의 큰 전환점
- H3는 H2 아래 설명 분기
- H4는 정말 필요할 때만 사용

권장 H2 개수:

- 짧은 글: 2~3개
- 일반 글: 3~5개
- 긴 해설 글: 5~7개

### 5-3. 리스트

리스트가 잘 맞는 경우:

- 기능 요약
- 비교 포인트
- 주의사항
- 활용 팁

문단이 더 나은 경우:

- 관점 설명
- 브랜드 철학
- 논리 전개

### 5-4. 강조

강조는 아래 순서로만 쓴다.

1. 소제목으로 구조 강조
2. 굵게 표시로 핵심어 강조
3. callout으로 중요한 안내 강조

밑줄, 색상 남발, 과도한 볼드는 피한다.

## 6. 컴포넌트 사용 기준

### 6-1. 이미지

현재 공통 UX에서 이미지는 클릭 확대가 가능하므로, 이미지는 "읽는 자료"이자 "보여주는 자료" 둘 다 될 수 있다.

원칙:

- 캡처 이미지는 글의 주장과 직접 연결된 것만 사용
- 장식용 이미지는 최소화
- 이미지 한 장당 전달 메시지가 명확해야 함
- 썸네일과 본문 첫 이미지가 완전히 중복되면 본문 쪽은 생략 검토

권장:

- 가로형 비율 우선
- 본문 중간 이미지에는 간단한 설명 문장 함께 배치

### 6-2. Callout

현재 callout은 매우 좋은 공통 문법이다.

권장 용도:

- info: 개념 설명, 보충 설명
- success: 결론, 추천 활용법
- warning: 주의사항, 오해 방지
- error: 문제 사례, 금지 또는 경고

원칙:

- 한 섹션에 1개면 충분
- callout만 여러 개 연속 배치하지 않는다
- 중요한 것만 박스로 빼고 나머지는 본문으로 유지

### 6-3. TOC

TOC는 긴 글에만 쓴다.

권장 기준:

- H2가 4개 이상
- 스크롤 길이가 긴 글
- 학습/해설 글

비권장:

- 2~3개 소제목뿐인 짧은 글
- 공지성 짧은 글

### 6-4. 태그

태그는 분류보다 검색성과 연결성을 높이는 장치다.

권장 예시:

- `SNarGPT`
- `AI`
- `국어`
- `업데이트`
- `교육철학`

비권장:

- 너무 긴 문장형 태그
- 거의 모든 글에 붙는 무의미한 범용 태그만 반복

## 7. 카테고리 간 일관성을 만드는 핵심 규칙

여기부터가 가장 중요하다.

### 규칙 1. 제목 구조를 통일한다

- 정보형: `무엇이 바뀌었는가 + 누구에게 중요한가`
- 해설형: `문제 제기 + 관점`
- 자료형: `자료명 + 활용 포인트`

### 규칙 2. 도입부 길이를 통일한다

- 도입은 보통 2~3문단
- 너무 길면 기사 같고, 너무 짧으면 카드 설명의 확장판처럼 보인다

### 규칙 3. 대표 시각 요소를 한 번은 쓴다

- 썸네일
- 본문 대표 이미지
- callout
- 표/리스트/TOC

글마다 형식은 달라도 "구조화된 글"이라는 인상을 줘야 한다.

### 규칙 4. 끝맺음 방식을 통일한다

마지막 섹션은 아래 중 하나로 정리한다.

- 요약
- 활용 팁
- 다음 행동
- 관련 글 연결

### 규칙 5. 브랜드 문체를 유지한다

브랜드 문체는 아래와 같다.

- 친절하지만 가볍지 않다
- 전문적이지만 과시하지 않는다
- 학생/학부모/교육 현장을 실제로 이해하는 톤이다
- AI를 말해도 늘 학습 맥락으로 돌아온다

## 8. SNarGPT 글 전용 디자인 규칙

앞으로 우선 많이 쓰게 될 `SNarGPT` 관련 글은 아래 규칙을 따르는 것을 추천한다.

### 권장 구조

1. 무엇이 달라졌는가
2. 왜 중요한가
3. 실제 사용 예시
4. 이런 학생에게 특히 유용하다
5. 앞으로의 방향

### 권장 시각 요소

- 제품 화면 캡처 1~3장
- 업데이트 포인트 callout
- 사용 예시 bullet
- 이전 대비 변화 요약

### 권장 제목 형식

- `SNarGPT [기능명] 업데이트: 무엇이 어떻게 달라졌나`
- `SNarGPT로 [학습 영역]을 공부하면 좋은 이유`
- `SNarGPT [기능명] 가이드: 처음 쓰는 학생을 위한 안내`

### 피해야 할 것

- 기술 용어만 가득한 제목
- 기능 나열만 있고 학습 효용이 없는 본문
- 너무 스타트업 홍보문 같은 어조

SNarGPT 글은 "AI 제품 블로그"와 "교육 콘텐츠"의 중간 지점에 있어야 가장 잘 맞는다.

---

## 9. 이미지와 썸네일 운영 규칙

### 썸네일

현재 목록 카드 구조에서는 썸네일 영향력이 크다.

원칙:

- 한 장에 메시지 하나
- 작은 카드에서도 읽히는 굵은 정보만 넣기
- 텍스트를 너무 많이 넣지 않기
- 브랜드 톤은 유지하되 카테고리별 변주를 주기

권장 방향:

- `startup`: 기술적이되 복잡하지 않은 UI/그래프형
- `columns`: 메시지형, 개념형, 비유형
- `datalab`: 데이터/지도/지표형
- `problem-download`: 문제지/해설/자료형
- `originals`: 작품명 + 상징 이미지형
- `notice`: 날짜/변경/안내가 한눈에 보이는 정보형

### 파일 관리

이미지 폴더도 카테고리 중심으로 유지하는 것이 좋다.

- `public/images/startup/...`
- `public/images/columns/...`
- `public/images/datalab/...`
- `public/images/notice/...`

## 10. 모바일 최적화 기준

현재 전역 스타일에서 한글 줄바꿈 보호를 잘 하고 있으므로, 작성 단계에서는 아래만 신경 쓰면 된다.

- 너무 긴 제목은 둘로 끊어도 자연스러운 구조로 쓴다
- 문단 첫 문장이 너무 길지 않게 한다
- 표는 꼭 필요한 경우만 사용한다
- 긴 URL은 링크 텍스트로 감춘다
- 이미지 속 글씨는 모바일에서도 읽히는 크기로 만든다

모바일에서 특히 중요한 것은 "문장 길이"와 "정보 덩어리 크기"다.

## 11. 작성 금지 패턴

아래 패턴은 블로그의 일관성을 가장 많이 해친다.

1. 제목만 강하고 본문 구조가 없는 글
2. 색상 강조가 지나친 글
3. callout 남용
4. 이미지 설명 없이 캡처만 많은 글
5. 문단이 지나치게 긴 글
6. 카테고리 성격과 맞지 않는 어조
7. 광고문처럼 보이는 과장된 표현

## 12. 앞으로의 운영 제안

실무적으로는 아래 3단계로 운영하면 좋다.

### A. 폴더 기준은 카테고리로 유지

현재처럼 아래 폴더 체계를 유지한다.

- `startup`
- `columns`
- `datalab`
- `problem-download`
- `originals`
- `notice`

### B. 시리즈 기준은 메타데이터로 관리

예:

```ts
series: 'snargpt'
```

이렇게 하면 `startup` 안에서 SNarGPT 시리즈를 계속 묶을 수 있다.

### C. 작성 전 체크리스트를 공통화

발행 전 아래 7가지를 확인한다.

1. 제목이 명확한가
2. excerpt가 클릭 이유를 주는가
3. 첫 3스크롤 안에 핵심이 보이는가
4. H2 구조가 정리되어 있는가
5. 이미지 또는 callout이 필요한 곳에만 들어갔는가
6. 태그가 적절한가
7. 모바일에서 읽히는가

## 13. 한 줄 결론

`snblog`의 디자인은 "미니멀한 교육 콘텐츠 레이아웃 + 브랜드 딥그린 포인트 + 구조화된 정보 전달"로 정의하는 것이 가장 정확하다.

앞으로는 카테고리별 폴더는 유지하되, 글 디자인은 아래 원칙으로 통일하면 된다.

- 레이아웃은 절제
- 본문은 가독성 우선
- 강조는 callout과 소제목 위주
- 이미지는 설명용으로만 사용
- 각 카테고리는 톤만 다르게, 구조는 비슷하게

이 기준이면 `SNarGPT` 글을 많이 써도 전체 블로그가 한 브랜드 안에서 안정적으로 보일 가능성이 높다.
